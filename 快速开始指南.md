## 🎯 核心功能

### 自动保存
每次修改数据后，系统会自动：
- 保存到浏览器本地（localStorage）
- 保存到云端（Supabase）
- 显示保存成功通知

### 自动加载
打开页面时，系统会：
1. 优先从云端加载数据
2. 如果云端没有，使用本地数据
3. 如果本地也没有，使用默认数据

### 手动同步
页面上有两个按钮：
- **☁️ 保存到云端**：手动触发保存
- **📥 从云端加载**：手动重新加载

## ⚠️ 常见问题

### Q1: 提示"保存失败"怎么办？

**检查清单：**
- [ ] 数据库表是否已创建？
- [ ] 表名是否为 `fund_data`？
- [ ] 列名是否正确？
- [ ] 网络连接是否正常？

**解决方法：**
1. 打开浏览器控制台查看错误信息
2. 参考 `Supabase数据库表创建指南.md` 检查表结构
3. 确认 Supabase 项目状态

### Q2: 数据没有自动同步？

**可能原因：**
- 不同设备使用了不同的用户ID
- 网络连接问题

**解决方法：**
1. 确认用户ID是否一致
2. 手动点击 "从云端加载" 按钮
3. 检查 Supabase 控制台是否有数据

### Q3: 如何重置数据？

**方法1：清空云端数据**
1. 访问 Supabase 控制台
2. 进入 Table Editor → fund_data
3. 删除你的数据行

**方法2：更换用户ID**
```javascript
// 在控制台执行
localStorage.removeItem('userId');
location.reload();
```
